What this is

A causal classifier that predicts, at every silence pause inside a user turn, the probability the turn is actually over (p_eot). Fed into the provided live-agent simulator, it decides how long a voice agent should wait before responding — the goal is to cut response latency without interrupting users mid-thought.

Method

Every pause gets a 14-dimensional causal feature vector computed only from audio[0:pause_start] (plus already-resolved earlier pauses in the same turn — never anything from the current or a future pause's pause_end):

  • Pitch (F0): trailing level, slope over the last 500ms, and level relative to the turn-so-far mean — falling pitch reads as a completed statement, level/rising pitch reads as a continuation.
  • Energy: trailing level, slope, and decay across the analysis window.
  • Voicing dynamics: voiced fraction (window + last 500ms), final-voiced-run length relative to the turn's average run (final-syllable lengthening), and voiced-run rate (a speaking-rate proxy).
  • Turn context: pause position in the turn, elapsed turn time, mean duration of this turn's earlier pauses, and an energy z-score against the turn's own baseline.

A gradient-boosted tree classifier (150 shallow trees) is trained on English and Hindi pooled together — the features are prosodic, not lexical, and the hidden test set is described as mostly Hindi, so the model shouldn't specialize to one language's vocabulary. Model selection used grouped (by-turn) cross-validation scored with the actual competition metric — mean delay at ≤5% cutoffs — not accuracy, since accuracy on an imbalanced hold/eot split doesn't track the metric that's actually graded.

Why it beats the status quo

The silence-only baseline (naive VAD endpointing: fire after a fixed timer, no exceptions) has no way to tell a genuine end of turn from a mid-sentence breath apart from waiting it out — its only lever is the timer, so it's stuck near the 1.6s timeout whenever hold-pauses run long enough to make short timers unsafe (English). Adding real prosodic signal — mainly the pitch-fall cue found in exploratory analysis — lets the policy fire confidently and early on true turn-ends while still recognizing "this pause looks like a mid-thought pause" from pitch/energy shape alone, cutting mean delay 54% on English and 23% on Hindi at the same 5% false-cutoff budget.

Where it still fails

Error analysis (error_analysis.py) found the worst misclassifications cluster on short, single-pause turns with rising final pitch that are nonetheless true end-of-turns — likely turn-final confirmation questions ("...for seven, right?"). These contradict the falling-pitch pattern the model mostly learned from longer, multi-pause turns, and with only ~250 pauses per language there isn't much data to learn the exception cleanly. See NOTES.md for what a next iteration would target.
